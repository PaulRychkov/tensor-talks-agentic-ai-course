# TensorTalks

## Проект

TensorTalks — AI-симулятор технических ML-собеседований. Платформа моделирует реальные интервью с AI-интервьюером, анализирует ответы и выдаёт персонализированный разбор: сильные стороны, слабые места и рекомендации по подготовке.

Основная аудитория MVP — ML-инженеры уровня junior-middle, которые готовятся к интервью в продуктовые компании. Проблема: специалисты месяцами готовятся, но не понимают, где именно проваливаются. Даже если есть отказ, обратная связь обычно формальная и не превращается в план действий. TensorTalks решает это — проводит интервью по программе, оценивает каждый ответ и формирует конкретный итог с ошибками и материалами.

На следующем этапе платформа также может использоваться компаниями для стандартизации оценки кандидатов и вузами как практический тренажёр.

Проект ВКР, работает на стыке AI-EdTech и HR-Tech.

## 4 этапа для применения агентов

| Этап | Текущее состояние | Что добавляем |
|------|-------------------|---------------|
| 1. Наполнение базы знаний и вопросов | Полностью вручную | LLM-workflow с внешними инструментами (`web_search`, `fetch_url`) для поиска и обработки ML-материалов. Human-in-the-loop для подтверждения |
| 2. Формирование интервью | Детерминированная сборка по фильтрам | Агент-планировщик: валидация программы, проверка покрытия тем, балансировка сложности |
| 3. Процесс интервью | LLM-workflow (фиксированная цепочка) | Агент-интервьюер: автономное ведение диалога с инструментами (оценка ответа, поиск теории в БД, `web_search`, суммаризация) |
| 4. Отчёт и материалы | Score + простой текстовый фидбек, подбора материалов нет | Агент-аналитик: анализ результатов, подбор материалов (БД + `web_search`), структурированный отчёт с планом подготовки |

## Текущее состояние

LLM используется только в одном из четырёх процессов — в процессе интервью. Реализован LLM-workflow: фиксированная цепочка шагов (оценка ответа, выбор действия, генерация реплики), где LLM вызывается по заданному сценарию. Система не планирует действия самостоятельно и не выбирает инструменты.

Остальные три этапа работают без LLM:
- **Наполнение базы знаний и вопросов** — полностью ручное
- **Сборка программы интервью** — детерминированная (выбор вопросов из базы по фильтрам)
- **Отчёт и подбор материалов** — минимальное (оценка + простой текстовый фидбек, подбора материалов нет)

## Что делаем

Превращаем LLM-workflow в агентную систему с инструментами (включая внешние), памятью и автономным поведением.

**Архитектура**:
- 3 агента на LangGraph (планировщик, интервьюер, аналитик)
- LLM-workflow для наполнения базы (детерминированный процесс)
- State & Memory: Redis (краткосрочная) + PostgreSQL (долгосрочная)
- Оркестрация: Kafka (асинхронное взаимодействие)
- Guardrails: pre-call и post-call проверки, лимиты, валидация

**Инструменты агентов**:
- Read-only: `search_questions`, `evaluate_answer`, `search_knowledge_base`, `get_evaluations`, `summarize_dialogue`, `check_topic_coverage`, `validate_program`, `get_topic_relations`
- Внешние: `web_search`, `fetch_url`
- Write (human-in-the-loop): `save_draft_material`
