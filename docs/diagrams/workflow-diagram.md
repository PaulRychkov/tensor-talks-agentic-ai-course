# Workflow Diagram

## Пошаговое выполнение запроса (Interview Flow)

Соответствие нумерации: **этап 0** в `docs/system-design.md` — LLM-workflow **Knowledge-producer** (наполнение базы, фоново), в эту схему не входит. Ниже — этапы 1–4.

```
┌──────────────────────────────────────────────────────────────────────────┐
│                        Interview workflow (этапы 1–4)                    │
└──────────────────────────────────────────────────────────────────────────┘

┌─────────┐
│ Start   │
└────┬────┘
     │
     ▼
┌─────────────────────────────────────────────────────────────────────────┐
│ Этап 1: Инициализация сессии (без LLM)                                  │
├─────────────────────────────────────────────────────────────────────────┤
│ 1. Пользователь задаёт параметры (например specialty, grade,            │
│    use_previous_results для интервью)                                   │
│ 2. Frontend → BFF: POST /api/sessions                                   │
│ 3. BFF → Session-service: создать сессию                                │
│ 4. Session-service → Redis: кеш                                         │
│ 5. Session-service → Kafka: interview.build.request                     │
│ 6. Session-service → Session-crud: метаданные сессии (программа позже)  │
│ 7. Session-service → BFF: сессия создана, ожидание программы            │
└─────────────────────────────────────────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────────────────────────────────────────┐
│ Этап 2: Формирование программы (Interview-builder, LangGraph)           │
├─────────────────────────────────────────────────────────────────────────┤
│ 1. Consume: interview.build.request                                     │
│ 2. Tools: search_questions, check_topic_coverage, validate_program,     │
│    get_topic_relations, search_knowledge_base (по режиму A/B/C)         │
│ 3. Publish: interview.build.response                                    │
│ 4. Session-crud: сохранить программу (или учебный план для study)       │
└─────────────────────────────────────────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────────────────────────────────────────┐
│ Этап 3: Проведение интервью (цикл по n вопросам, Agent Interviewer)     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│ ┌─────────────────────────────────────────────────────────────────────┐ │
│ │ Цикл для каждого из n вопросов:                                     │ │
│ │                                                                     │ │
│ │  ┌───────────────────────────────────────────────────────────────┐  │ │
│ │  │ Шаг 1: Формулирование вопроса                                 │  │ │
│ │  │ Agent-интервьюер → LangGraph State: загрузить контекст        │  │ │
│ │  │ Agent → LLM: сформулировать вопрос                            │  │ │
│ │  │ Agent → Kafka: generated.phrases (вопрос)                     │  │ │
│ │  │ Dialogue-aggregator → Kafka: chat.events.in                   │  │ │
│ │  │ BFF → Frontend: доставить вопрос                              │  │ │
│ │  └───────────────────────────────────────────────────────────────┘  │ │
│ │                            │                                        │ │
│ │                            ▼                                        │ │
│ │  ┌───────────────────────────────────────────────────────────────┐  │ │
│ │  │ Шаг 2: Ожидание ответа                                        │  │ │
│ │  │ Пользователь → Frontend: ввод ответа                          │  │ │
│ │  │ Frontend → BFF: POST /api/sessions/{id}/message               │  │ │
│ │  │ BFF → Kafka: chat.events.out (сообщение)                      │  │ │
│ │  │ Dialogue-aggregator → Chat-crud: сохранить сообщение          │  │ │
│ │  │ Dialogue-aggregator → Kafka: messages.full.data               │  │ │
│ │  └───────────────────────────────────────────────────────────────┘  │ │
│ │                            │                                        │ │
│ │                            ▼                                        │ │
│ │  ┌───────────────────────────────────────────────────────────────┐  │ │
│ │  │ Шаг 3: Анализ ответа (Агент)                                  │  │ │
│ │  │ Agent читает Kafka: messages.full.data                        │  │ │
│ │  │ Agent → Tool: evaluate_answer(question, answer, theory)       │  │ │
│ │  │ Tool → LLM: оценка по критериям                               │  │ │
│ │  │ Tool → Agent: Pydantic AnswerEvaluation                       │  │ │
│ │  │   (score, decision_confidence, errors, missing_points)        │  │ │
│ │  └───────────────────────────────────────────────────────────────┘  │ │
│ │                            │                                        │ │
│ │                            ▼                                        │ │
│ │  ┌───────────────────────────────────────────────────────────────┐  │ │
│ │  │ Шаг 4: Принятие решения                                       │  │ │
│ │  │ Agent анализирует оценку:                                     │  │ │
│ │  │                                                               │  │ │
│ │  │ ┌──────────────────────────────────────────────────────────┐  │  │ │
│ │  │ │ Ответ полный (score >= 0.85) + confidence >= 0.7         │  │  │ │
│ │  │ │ → Перейти к следующему вопросу                           │  │  │ │
│ │  │ └──────────────────────────────────────────────────────────┘  │  │ │
│ │  │ ┌──────────────────────────────────────────────────────────┐  │  │ │
│ │  │ │ Низкая уверенность (confidence < 0.5)                    │  │  │ │
│ │  │ │ → self-reflection: переоценка с доп. контекстом          │  │  │ │
│ │  │ └──────────────────────────────────────────────────────────┘  │  │ │
│ │  │ ┌──────────────────────────────────────────────────────────┐  │  │ │
│ │  │ │ Ответ частичный (0.4 <= score < 0.85) или conf < 0.7     │  │  │ │
│ │  │ │ → search_knowledge_base, search_questions(related_to)    │  │  │ │
│ │  │ │ → подсказка / подвопрос (генерирует LLM)                 │  │  │ │
│ │  │ └──────────────────────────────────────────────────────────┘  │  │ │
│ │  │ ┌──────────────────────────────────────────────────────────┐  │  │ │
│ │  │ │ Off-topic                                                │  │  │ │
│ │  │ │ → Вернуть к теме                                         │  │  │ │
│ │  │ └──────────────────────────────────────────────────────────┘  │  │ │
│ │  │ ┌──────────────────────────────────────────────────────────┐  │  │ │
│ │  │ │ "Не знаю"                                                │  │  │ │
│ │  │ │ → Подсказка с опорой на теорию                           │  │  │ │
│ │  │ └──────────────────────────────────────────────────────────┘  │  │ │
│ │  │ ┌──────────────────────────────────────────────────────────┐  │  │ │
│ │  │ │ Запрос пропуска                                          │  │  │ │
│ │  │ │ → Записать в оценку, перейти дальше                      │  │  │ │
│ │  │ └──────────────────────────────────────────────────────────┘  │  │ │
│ │  │ ┌──────────────────────────────────────────────────────────┐  │  │ │
│ │  │ │ Свежий фреймворк (по необходимости)                      │  │  │ │
│ │  │ │ → web_search + fetch_url                                 │  │  │ │
│ │  │ └──────────────────────────────────────────────────────────┘  │  │ │
│ │  └───────────────────────────────────────────────────────────────┘  │ │
│ │                            │                                        │ │
│ │                            ▼                                        │ │
│ │  ┌───────────────────────────────────────────────────────────────┐  │ │
│ │  │ Шаг 5: Генерация реплики                                      │  │ │
│ │  │ Agent → LLM: сгенерировать ответ                              │  │ │
│ │  │ Agent → Kafka: generated.phrases                              │  │ │
│ │  │ Dialogue-aggregator → Chat-crud: сохранить ответ              │  │ │
│ │  │ Dialogue-aggregator → Kafka: chat.events.in                   │  │ │
│ │  │ BFF → Frontend: доставить ответ                               │  │ │
│ │  └───────────────────────────────────────────────────────────────┘  │ │
│ │                                                                     │ │
│ └─────────────────────────────────────────────────────────────────────┘ │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
     │ (после n вопросов)
     ▼
┌─────────────────────────────────────────────────────────────────────────┐
│ Этап 4: Отчёт и пресеты (Agent Analyst)                                 │
├─────────────────────────────────────────────────────────────────────────┤
│ 1. Dialogue-aggregator → Kafka: session.completed (session_id,          │
│    session_kind, topics, level)                                         │
│ 2. Analyst-agent-service consume → LangGraph, маршрутизация по          │
│    session_kind, подгрузка state / Chat-crud / Results                  │
│ 3. get_evaluations; при необходимости уточнение оценок (LLM)            │
│ 4. group_errors_by_topic                                                │
│ 5. Материалы: search_knowledge_base(query, topic); при нехватке —       │
│    web_search, fetch_url                                                │
│ 6. При необходимости: save_draft_material (HITL), без автозаписи в KB   │
│ 7. generate_report_section (Summary, Errors, Strengths, Plan, Materials)│
│ 8. validate_report → при FAIL повторить доработку (лимит итераций)      │
│ 9. Results-crud: отчёт + preset_training                                │
│ 10. Kafka chat.events.in: отчёт пользователю                            │
│ 11. BFF → Frontend: /results (тренировки / study)                       │
└─────────────────────────────────────────────────────────────────────────┘
     │
     ▼
┌─────────┐
│  End    │
└─────────┘
```

## Ветки ошибок (Error Branches)

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    Error Handling Branches                              │
└─────────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────┐
│ LLM Timeout (>30 секунд)                                             │
├──────────────────────────────────────────────────────────────────────┤
│ 1. Детект: TimeoutException при вызове LLM                           │
│ 2. Retry (max 2 попытки с экспоненциальной задержкой)                │
│ 3. Если всё ещё таймаут:                                             │
│    → Fallback: "Извините, техническая задержка. Продолжаем..."       │
│    → Переход к следующему вопросу                                    │
│ 4. Логирование инцидента                                             │
└──────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────┐
│ Невалидный JSON от LLM                                               │
├──────────────────────────────────────────────────────────────────────┤
│ 1. Детект: JSONDecodeError при парсинге оценки                       │
│ 2. Retry с тем же промптом (1 попытка)                               │
│ 3. Если всё ещё невалидный:                                          │
│    → Шаблонная оценка: { score: 0.5, manual_review: true }           │
│    → Переход к следующему вопросу                                    │
│ 4. Логирование для последующего анализа                              │
└──────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────┐
│ Зацикливание (N > 5 tool calls на шаг)                               │
├──────────────────────────────────────────────────────────────────────┤
│ 1. Детект: счётчик вызовов инструментов > 5                          │
│ 2. Принудительный переход к следующему действию                      │
│ 3. Логирование: "Agent loop detected, forcing progression"           │
└──────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────┐
│ web_search возвращает нерелевантное                                  │
├──────────────────────────────────────────────────────────────────────┤
│ 1. Детект: агент оценивает релевантность < 0.5                       │
│ 2. Retry с другим query (max 3 попытки)                              │
│ 3. Если всё ещё нерелевантно:                                        │
│    → Отказ от внешнего поиска                                        │
│    → Использование только внутренних материалов                      │
│    → Пометка в отчёте: "Внешние материалы не найдены"                │
└──────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────┐
│ Перерасход токенов (>8000)                                           │
├──────────────────────────────────────────────────────────────────────┤
│ 1. Детект: context_budget < 0                                        │
│ 2. Автоматическая суммаризация: summarize_dialogue()                 │
│ 3. Предупреждение пользователю: "Длинный диалог, сжимаем контекст"   │
│ 4. Продолжение с обновлённым бюджетом                                │
└──────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────┐
│ Отказ Kafka (ConnectionError)                                        │
├──────────────────────────────────────────────────────────────────────┤
│ 1. Детект: Kafka connection error                                    │
│ 2. Буферизация сообщений в памяти (max 100)                          │
│ 3. Retry с экспоненциальной задержкой (1s, 2s, 4s, 8s, 16s)          │
│ 4. При восстановлении: отправка буферизованных сообщений             │
│ 5. Если не восстановлено > 1 минуты:                                 │
│    → Fallback: прямое HTTP взаимодействие (деградация)               │
│    → Алерт в мониторинг                                              │
└──────────────────────────────────────────────────────────────────────┘
```

## Состояния агента (State Transitions)

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    Agent State Machine                                  │
└─────────────────────────────────────────────────────────────────────────┘

     ┌─────────────┐
     │  INITIALIZED│
     └──────┬──────┘
            │ (interview.build.request → response)
            ▼
     ┌─────────────┐
     │  BUILDING_  │◀──┐
     │   PROGRAM   │───┘ (retry: validate_program / coverage)
     └──────┬──────┘
            │ (программа в state)
            ▼
     ┌─────────────┐
     │ INTERVIEW_  │
     │   ACTIVE    │
     └──────┬──────┘
            │ (цикл по вопросам)
            │
     ┌──────┴──────┐
     │             │
     │             │ (все n вопросов)
     │             │
     ▼             ▼
┌─────────┐  ┌────────────┐
│ SKIPPED │  │ COMPLETED  │
└─────────┘  └─────┬──────┘
                   │ (формирование отчёта)
                   ▼
            ┌─────────────┐
            │  ANALYZING  │
            └──────┬──────┘
                   │ (отчёт готов)
                   ▼
            ┌─────────────┐
            │   SAVED     │
            └─────────────┘
```
